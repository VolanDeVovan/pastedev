chapter 09

Shared components

Four building blocks every page reaches for: Shell (top bar + mobile tab bar), Modal, ToastDock, FormField.

Shell — top bar + bottom tab bar

The Shell wraps page content with a 52-pixel <header> and reserves space for the mobile bottom tab bar (pb-[68px] md:pb-0). The TopBar shows nav links only for approved users; the bottom bar only shows up for approved users on mobile.

Dioxus 0.7 supports children-as-a-default-slot — equivalent to Vue's <slot/>: 

#[component]
pub fn Shell(children: Element) -> Element {
    let auth = use_auth();
    let route = use_route::<Route>();

    rsx! {
        div { class: "min-h-screen flex flex-col",
            header { class: "border-b border-border",
                div { class: "max-w-6xl mx-auto px-4 md:px-7 h-[52px] flex items-center gap-3 md:gap-6 text-sm",
                    Link { to: Route::Editor { edit: None },
                        class: "font-bold tracking-tight text-text", "pastedev" }
                    if auth.is_approved() {
                        nav { class: "hidden md:flex gap-6 ml-1",
                            NavLink { to: Route::Editor { edit: None }, active: matches!(route, Route::Editor { .. }), "new" }
                            NavLink { to: Route::Dashboard {}, active: matches!(route, Route::Dashboard {}), "my snippets" }
                            NavLink { to: Route::ApiKeys {},  active: matches!(route, Route::ApiKeys {}),  "api keys" }
                            if auth.is_admin() {
                                NavLink { to: Route::Admin {}, active: matches!(route, Route::Admin {}), "admin" }
                            }
                        }
                    }
                    div { class: "ml-auto flex items-center gap-2 md:gap-3 text-xs",
                        UserMenu {}
                    }
                }
            }
            main { class: "flex-1 pb-[68px] md:pb-0", {children} }
            if auth.is_approved() { MobileTabBar { route } }
        }
    }
}

Modal — teleport + escape

Vue's Teleport to="body" matches Dioxus's Portal primitive — modal markup is rendered into a <div id="portal-root"> we add to index.html, keeping it out of the page DOM tree but with the right z-index. Escape closes it; clicking the backdrop closes it; clicking the panel does not. 

#[component]
pub fn Modal(
    open:    Signal<bool>,
    title:   Option<String>,
    danger:  Option<bool>,
    children: Element,
    actions:  Option<Element>,
    on_confirm: EventHandler,
) -> Element {
    use_keyboard_escape(move || open.set(false));

    if !open() { return rsx! {}; }
    rsx! {
        Portal {
            div {
                class: "fixed inset-0 z-50 flex items-center justify-center bg-bg/80 backdrop-blur-sm",
                onmousedown: move |e| if e.target() == e.current_target() { open.set(false) },
                div {
                    class: "bg-bg-deep border border-border-strong rounded-sm w-[440px] max-w-[calc(100vw-2rem)] relative overflow-hidden",
                    onmousedown: |e| e.stop_propagation(),
                    span { class: if danger.unwrap_or(false) { "absolute left-0 top-0 bottom-0 w-px bg-danger" } else { "absolute left-0 top-0 bottom-0 w-px bg-accent" } }
                    div { class: "flex items-start justify-between px-5 pt-4 pb-3 border-b border-border",
                        div { class: "text-[13px] text-text", if let Some(t) = title.as_ref() { "{t}" } }
                        button { onclick: move |_| open.set(false), "×" }
                    }
                    div { class: "px-5 py-4 text-[12px] text-text-dim leading-relaxed", {children} }
                    div { class: "flex justify-end items-center gap-2 px-4 py-3 border-t border-border bg-bg/40",
                        if let Some(slot) = actions {
                            {slot}
                        } else {
                            button { onclick: move |_| open.set(false), "cancel" }
                            button { onclick: move |_| on_confirm.call(()), "confirm" }
                        }
                    }
                }
            }
        }
    }
}

The CSS transition (.modal-enter / .modal-leave) becomes a Tailwind utility on the wrapper — same opacity ramp as today, driven by data-attribute and CSS rather than a Vue Transition.

ToastDock

A fixed bottom-right column that reads the toast queue and renders each item with a per-kind border. transition-group becomes a CSS rule keyed off data-state="enter"|"leave"; Dioxus's use_effect can flip the state on mount and schedule the removal-state right before dismissal.

FormField

Just a labelled input or textarea with the same Tailwind classes. The Vue version used useAttrs to forward arbitrary attributes; Dioxus props are explicit, so we list what we actually need and call it a day. 

#[derive(Props, PartialEq, Clone)]
pub struct FormFieldProps {
    label: String,
    value: Signal<String>,
    #[props(default)] hint:         Option<String>,
    #[props(default)] placeholder:  Option<String>,
    #[props(default)] kind:         Option<String>,  // "text" | "password" | "email"
    #[props(default)] autocomplete: Option<String>,
    #[props(default)] required:     bool,
    #[props(default)] rows:         Option<u32>,
}

#[component]
pub fn FormField(props: FormFieldProps) -> Element {
    let id = use_hook(|| format!("f-{}", slug(&props.label)));
    rsx! {
        label { r#for: "{id}", class: "block mb-3.5",
            div { class: "flex justify-between items-baseline mb-1.5",
                span { class: "text-[11px] text-text-muted", "{props.label}" }
                if let Some(h) = &props.hint { span { class: "text-[10px] text-text-faint", "{h}" } }
            }
            if let Some(rows) = props.rows {
                textarea { id, rows, value: "{props.value}",
                    oninput: move |e| props.value.set(e.value()),
                    // ... classes ...
                }
            } else {
                input { id, r#type: props.kind.as_deref().unwrap_or("text"),
                    value: "{props.value}", autocomplete: props.autocomplete.as_deref(),
                    required: props.required,
                    oninput: move |e| props.value.set(e.value()),
                    // ... classes ...
                }
            }
        }
    }
}

Tiny helpers worth pulling out

  • CenteredCard { width: "..."} — the flex justify-center pt-32 wrapper used by signin/register/pending/rejected.
  • NavLink — wraps Link with the active-underline classes.
  • EmptyRow / ErrorRow / LoadingRow — single-line rows for "no snippets yet" / "loading…" / "{error.message()}". Avoids repeating the same div in five pages.
  • TimeAgo { iso: String } — replaces the four copies of function ago(iso) scattered across views.

Icons

The app uses ≈8 inline glyph characters (›, ×, ↗, ↵, etc.) and no SVG icon set. We keep them as string literals — no icon library to introduce.

Components we don't bring across

  • HomeView.vue — already redundant in Vue; the root route renders the editor. We drop the file.
  • Vue Transition wrappers — replaced with CSS transitions keyed off mount/state. The visual result is identical.
  • Pinia stores — see chapter 04, replaced by context + signals.
