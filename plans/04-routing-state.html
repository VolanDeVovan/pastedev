Routing & state

All 13 routes, three guard layers, and two stores — translated into a single Routable enum and two contexts.

The route table

Same paths, same names, same guard rules as web/src/router.ts. What changes is that the guard logic moves from imperative router.beforeEach callbacks into layout components — a tiny wrapper component sits above each protected branch and conditionally renders Outlet or redirects.

parity-critical: setup gate covers public read paths too
Today's router.beforeEach redirects every route except /setup and /status to /setup while needs_setup is true — including /c/:slug, /m/:slug, /h/:slug. The route enum below reflects that: every variant sits inside #[layout(RootLayout)], which owns the boot + setup-gate. Per-area auth checks layer underneath.

The Route enum

// crates/web/src/route.rs
use dioxus::prelude::*;
use crate::{pages::*, views::*, editor::EditorPage, components::*};

#[derive(Routable, Clone, PartialEq)]
#[rustfmt::skip]
pub enum Route {
    // RootLayout owns the universal contract:
    //   1. Run auth.boot() exactly once per session (refresh_me + refresh_setup).
    //   2. While loading, render <SplashFallback />.
    //   3. While needs_setup is true, redirect to /setup from EVERY route
    //      except /setup and /status. This matches today's router.beforeEach
    //      and covers public slug routes too — without it /c/:slug would
    //      load on a fresh instance, which is a parity regression.
    #[layout(RootLayout)]
        // Always reachable, even when needs_setup, even when signed out.
        #[route("/status")]
        Status {},

        // Bootstrap-only — RootLayout bounces here while needs_setup is true,
        // and bounces away from here once it's false.
        #[route("/setup")]
        Setup {},

        // Public read-only views — no auth required, but setup-gated by
        // virtue of sitting inside RootLayout.
        #[route("/c/:slug")]  ViewCode { slug: String },
        #[route("/m/:slug")]  ViewMarkdown { slug: String },
        #[route("/h/:slug")]  ViewHtml { slug: String },

        // AuthedShell layers in the signed-out vs signed-in shaping
        // (bounce /signin while signed-in, park pending/rejected users).
        // Slug routes deliberately sit ABOVE it — anonymous visitors must
        // be able to read them without going through this guard.
        #[layout(AuthedShell)]
            #[route("/signin?:next")]
            SignIn { next: Option<String> },

            #[route("/register")]
            Register {},

            #[route("/pending")]
            Pending {},

            #[route("/rejected")]
            Rejected {},

            // Admin-only.
            #[layout(RequireAdmin)]
                #[route("/admin")]  Admin {},
            #[end_layout]

            // Approved-only.
            #[layout(RequireApproved)]
                #[route("/dashboard")]  Dashboard {},
                #[route("/keys")]       ApiKeys {},
                #[route("/?:edit")]     Editor { edit: Option<String> },
            #[end_layout]
        #[end_layout]

        #[route("/:..rest")]
        NotFound { rest: Vec<String> },
    #[end_layout]
}

Guard layouts

Four wrapper components do the work the old async beforeEach did, top to bottom: RootLayout (boot + setup-gate), AuthedShell (signed-in / pending / rejected shaping), RequireApproved, RequireAdmin. Each reads use_auth() and either renders Outlet or schedules a redirect through navigator().replace(...).

RootLayout — boot & setup-gate, applies to everything

#[component]
pub fn RootLayout() -> Element {
    let auth  = use_auth();
    let route = use_route::<Route>();
    let nav   = navigator();

    // First nav of the session: kick off boot. use_effect with a stable
    // closure ensures it fires exactly once, regardless of which child
    // component reads from auth signals on the first render.
    use_effect(move || {
        if auth.last_fetch_at.peek().is_none() {
            spawn(async move { auth.boot().await; });
        }
    });

    // Splash until the first boot completes. SplashFallback is just the
    // shell HTML with an empty body — same bytes the server already ships.
    if auth.last_fetch_at.read().is_none() {
        return rsx! { SplashFallback {} };
    }

    // Setup-gate: while needs_setup, only /status is reachable. Every
    // other route (including /c|m|h/:slug, /signin, /register, /) bounces
    // to /setup. Conversely, once setup is done, /setup itself bounces to
    // the editor. Matches today's beforeEach for parity.
    let in_setup     = matches!(route, Route::Setup {});
    let in_status    = matches!(route, Route::Status {});
    if auth.needs_setup() && !in_setup && !in_status {
        nav.replace(Route::Setup {});
        return rsx! { SplashFallback {} };
    }
    if !auth.needs_setup() && in_setup {
        nav.replace(Route::Editor { edit: None });
        return rsx! { SplashFallback {} };
    }

    rsx! { Outlet::<Route> {} }
}

AuthedShell — signed-in shaping

#[component]
pub fn AuthedShell() -> Element {
    let auth  = use_auth();
    let route = use_route::<Route>();
    let nav   = navigator();

    // Signed-in users can't sit on /signin or /register.
    if auth.user.read().is_some()
        && matches!(route, Route::SignIn { .. } | Route::Register {})
    {
        nav.replace(Route::Editor { edit: None });
        return rsx! { SplashFallback {} };
    }

    // Park pending / rejected users on their landing page.
    if let Some(user) = auth.user.read().as_ref() {
        match (user.status, &route) {
            (UserStatus::Pending,  r) if !matches!(r, Route::Pending {}) => {
                nav.replace(Route::Pending {});
                return rsx! { SplashFallback {} };
            }
            (UserStatus::Rejected, r) if !matches!(r, Route::Rejected {}) => {
                nav.replace(Route::Rejected {});
                return rsx! { SplashFallback {} };
            }
            _ => {}
        }
    }
    rsx! { Outlet::<Route> {} }
}

RequireApproved / RequireAdmin

#[component]
pub fn RequireApproved() -> Element {
    let auth = use_auth();
    let nav  = navigator();
    match auth.user.read().as_ref().map(|u| u.status) {
        Some(UserStatus::Approved) => rsx! { Outlet::<Route> {} },
        Some(_) => { nav.replace(Route::Pending {});  rsx! { SplashFallback {} } }
        None    => {
            let next = current_path();
            nav.replace(Route::SignIn { next: Some(next) });
            rsx! { SplashFallback {} }
        }
    }
}

#[component]
pub fn RequireAdmin() -> Element {
    let auth = use_auth();
    let nav  = navigator();
    if auth.is_admin() {
        rsx! { Outlet::<Route> {} }
    } else {
        nav.replace(Route::Editor { edit: None });
        rsx! { SplashFallback {} }
    }
}

// current_path() is a one-liner over web-sys; lives in src/lib/path.rs.
pub fn current_path() -> String {
    web_sys::window()
        .and_then(|w| w.location().pathname().ok().zip(w.location().search().ok()))
        .map(|(p, s)| format!("{p}{s}"))
        .unwrap_or_else(|| "/".into())
}
why layouts and not async guards
Dioxus has no router.beforeEach equivalent — and that's the right call. Async guards racing with the render cycle are a known source of "flash of wrong page" bugs in vue-router. A layout component just re-renders when state changes and picks the right child synchronously.
don't redirect during render
Calling navigator().replace(...) directly inside a component's body during the render pass is fine in Dioxus (the navigator queues the replace, no warning), but if you find yourself fighting render-time warnings, move the redirect into a use_effect that fires when the guard predicate changes. Returning SplashFallback (an empty shell) for the one frame between the decision and the navigation prevents flashes of the wrong page.

Auth state

A single struct of Signals, provided once at the root and grabbed by every component with use_auth(). The methods on it mirror what useAuthStore exposed today. 

// crates/web/src/state/auth.rs
#[derive(Clone, Copy)]
pub struct AuthState {
    pub user:           Signal<Option<UserPublic>>,
    pub setup:          Signal<Option<SetupStatus>>,
    pub loading:        Signal<bool>,
    pub last_fetch_at: Signal<Option<f64>>, // Date.now()
}

impl AuthState {
    pub fn needs_setup(&self) -> bool {
        self.setup.read().as_ref().is_some_and(|s| s.needs_setup)
    }
    pub fn is_admin(&self) -> bool {
        self.user.read().as_ref().is_some_and(|u| u.role == Role::Admin)
    }
    pub fn is_approved(&self) -> bool {
        self.user.read().as_ref().is_some_and(|u| u.status == UserStatus::Approved)
    }

    pub async fn boot(mut self) {
        self.loading.set(true);
        let (s, u) = futures::join!(api::auth::setup_status(), api::auth::me());
        self.setup.set(s.ok());
        self.user.set(u.ok());
        self.last_fetch_at.set(Some(now_ms()));
        self.loading.set(false);
    }
    pub async fn login(mut self, input: LoginInput) -> Result<(), HttpError> {
        let u = api::auth::login(input).await?;
        self.user.set(Some(u));
        Ok(())
    }
    pub async fn logout(mut self) -> Result<(), HttpError> { /* ... */ }
    pub async fn register(mut self, ...) -> Result<(), HttpError> { /* ... */ }
    pub async fn refresh_me(mut self) { /* ... */ }
    pub async fn refresh_setup(mut self) { /* ... */ }
    pub async fn setup_admin(mut self, input: CreateAdminInput) { /* ... */ }
}

pub fn provide_auth() -> AuthState {
    use_context_provider(|| AuthState {
        user:           Signal::new(None),
        setup:          Signal::new(None),
        loading:        Signal::new(false),
        last_fetch_at:  Signal::new(None),
    })
}

Toast state

Even smaller — one signal-of-vec, a couple of helpers, a TTL via gloo_timers::callback::Timeout: 

// crates/web/src/state/toast.rs
#[derive(Clone, PartialEq)]
pub struct Toast { pub id: u64, pub kind: ToastKind, pub message: String }

#[derive(Clone, Copy)]
pub struct ToastQueue { pub items: Signal<Vec<Toast>> }

impl ToastQueue {
    pub fn info(mut self, msg: impl Into<String>)    { self.push(ToastKind::Info, msg, 4000); }
    pub fn error(mut self, msg: impl Into<String>)   { self.push(ToastKind::Error, msg, 6000); }
    pub fn success(mut self, msg: impl Into<String>) { self.push(ToastKind::Success, msg, 4000); }
    fn push(&mut self, kind: ToastKind, msg: impl Into<String>, ttl_ms: u32) {
        let id = next_id();
        self.items.with_mut(|v| v.push(Toast { id, kind, message: msg.into() }));
        let mut items = self.items;
        gloo_timers::callback::Timeout::new(ttl_ms, move || {
            items.with_mut(|v| v.retain(|t| t.id != id));
        }).forget();
    }
}

Error-on-403 → refresh-me pattern

The Vue editor catches a 403 from createSnippet and calls auth.refreshMe() so a user who got demoted/suspended mid-session sees the right page on next navigation. We keep that behaviour. 

match api::snippets::create(input).await {
    Ok(s)  => nav.replace(Route::for_snippet(&s)),
    Err(e) => {
        if e.code() == "forbidden" { auth.refresh_me().await; }
        toast.error(e.message());
    }
}
