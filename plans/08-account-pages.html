Auth pages, admin queue, API keys, setup

Sign in

#[component]
pub fn SignIn(next: Option<String>) -> Element {
    let auth = use_auth();
    let nav  = navigator();
    let mut username = use_signal(String::new);
    let mut password = use_signal(String::new);
    let mut error    = use_signal::<Option<String>>(|| None);
    let mut submitting = use_signal(|| false);

    let submit = move |_| spawn(async move {
        submitting.set(true); error.set(None);
        match auth.login(LoginInput { username: username().trim().into(), password: password() }).await {
            Ok(_) => {
                let target = next.clone().unwrap_or_else(|| "/".into());
                nav.replace_str(&target);
            }
            Err(e) => error.set(Some(e.message().into())),
        }
        submitting.set(false);
    });

    rsx! { Shell { CenteredCard { width: "380px",
        h1 { class: "text-[22px] tracking-tight mb-1.5", "sign in" }
        p  { class: "text-[12px] text-text-muted leading-relaxed mb-7", "use your credentials to continue." }
        form { onsubmit: submit,
            FormField { label: "username", value: username, autocomplete: "username", required: true }
            FormField { label: "password", value: password, kind: "password", required: true }
            if let Some(e) = error() { div { class: "text-[12px] text-danger mb-3", "{e}" } }
            button { r#type: "submit", disabled: submitting(),
                class: "w-full bg-accent text-bg-deep font-semibold py-2.5 text-[13px] rounded-sm hover:opacity-90 disabled:opacity-30",
                if submitting() { "signing in…" } else { "continue →" }
            }
        }
    } } }
}

Register

Same shape, one extra field (reason textarea) and a client-side check that rejects bodies shorter than 10 chars before posting. After a successful register the user redirects to /pending.

Pending — polling

Translates the setInterval(poll, 10_000) into a use_future loop driven by gloo_timers: 

use_future(move || async move {
    loop {
        gloo_timers::future::TimeoutFuture::new(10_000).await;
        auth.refresh_me().await;
        if let Some(u) = auth.user.read().as_ref() {
            match u.status {
                UserStatus::Approved => { navigator().replace(Route::Editor { edit: None }); break; }
                UserStatus::Rejected => { navigator().replace(Route::Rejected {}); break; }
                _ => {}
            }
        }
    }
});

Status — adaptive cadence

Same idea but the wait is 5s when everything is OK, 15s otherwise. overall is a use_memo over the checks array; the loop just reads it before each await.

Setup wizard

Two-step flow (check → admin) with the same dot/label colors. Polls every 2 seconds in step 1 to see if the user fixed env vars (e.g. set PASTEDEV_SECRET). On submit it calls auth.setup_admin and redirects to /. The mobile vs desktop step strip is just CSS — keep it.

Dashboard

The largest page after the editor. Three things going on:

  1. One-shot fetch with limit=50, plus a load more link that fires the next page using next_cursor.
  2. Client-side filter tabs (all / code / md / html) over the loaded items.
  3. Free-text filter on name OR slug — the same predicate we use today.
let mut all     = use_signal::<Vec<SnippetListItem>>(Vec::new);
let mut next    = use_signal::<Option<String>>(|| None);
let mut filter  = use_signal(|| FilterTab::All);
let mut query   = use_signal(String::new);

let items = use_memo(move || {
    let q = query().to_lowercase();
    let q = q.trim();
    all.read().iter()
        .filter(|i| filter().matches(i.r#type))
        .filter(|i| q.is_empty()
            || i.name.as_deref().is_some_and(|n| n.to_lowercase().contains(q))
            || i.slug.to_lowercase().contains(q))
        .cloned()
        .collect::<Vec<_>>()
});

let counts = use_memo(move || {
    let items = all.read();
    FilterCounts {
        all:      items.len(),
        code:     items.iter().filter(|i| i.r#type == SnippetType::Code).count(),
        markdown: items.iter().filter(|i| i.r#type == SnippetType::Markdown).count(),
        html:     items.iter().filter(|i| i.r#type == SnippetType::Html).count(),
    }
});

Admin queue

Same six actions: approve / reject / suspend / restore / promote / demote. Approve and reject go through the confirmation Modal; the other four fire immediately. Avatar hue derives from a username hash: 

const PALETTE: &[&str] = &[
    "bg-accent/20 text-accent",
    "bg-warn/20 text-warn",
    "bg-blue-400/20 text-blue-300",
    "bg-rose-400/20 text-rose-300",
    "bg-purple-400/20 text-purple-300",
    "bg-emerald-400/20 text-emerald-300",
];
fn avatar_class(name: &str) -> &'static str {
    let h: u32 = name.bytes().fold(0u32, |a, b| a.wrapping_mul(31).wrapping_add(b as u32));
    PALETTE[(h as usize) % PALETTE.len()]
}

API keys

Two columns on desktop, one on mobile. Left column: list + generate-form + the one-shot "new key created — copy now" callout. Right column: install / authenticate / publish / MCP code blocks.

The install pane has a macOS·Linux / Windows toggle that defaults to Windows when navigator().user_agent() contains "windows". That works the same in Dioxus, just via web_sys::window().navigator().user_agent().

The install / authenticate / publish blocks substitute {instance_url} — the value of config().public_base_url at runtime. We render them with rsx! conditionals rather than the giant inline <pre> blobs the Vue code uses; the tokens ($, pds_live_…) become small <span>s with the same classes.

Forms — typed inputs

Every page above uses the FormField component (chapter 09). It centralizes label / hint / textarea-vs-input / autocomplete / aria so we don't repeat the Tailwind classes.

Polling: a tiny helper

Three pages (setup, status, pending) all want "run X every N ms while mounted, with N possibly changing over time, cancel on unmount". Same pattern as the Vue code (onMounted + onUnmounted + setInterval); we abstract it once: 

// crates/web/src/lib/polling.rs
pub fn use_poll<F, Fut>(every_ms: impl Fn() -> u32 + 'static, mut work: F)
where
    F: FnMut() -> Fut + 'static,
    Fut: Future<Output = ()> + 'static,
{
    use_future(move || async move {
        loop {
            work().await;
            gloo_timers::future::TimeoutFuture::new(every_ms()).await;
        }
    });
}

Error handling parity

  • Form submit errors land in a single error signal under the form fields.
  • Background errors (refresh polls, secondary calls) post to toast.error.
  • 403 on a state-changing call → auth.refresh_me().await before showing the error, so the guard layout reroutes the user on the next render. Today's behaviour, kept exact.
