chapter 05

API client

The Rust replacement for web/src/api/index.ts. One thin reqwest wrapper, one error type that mirrors the server's envelope, and one helper per endpoint.

The core client

// crates/web/src/api/client.rs
use reqwest::{Client, Method, RequestBuilder, Response, StatusCode};
use serde::{de::DeserializeOwned, Serialize};
use crate::config::config;
use crate::api::error::{ApiError, HttpError};

fn client() -> &'static Client {
    static C: OnceLock<Client> = OnceLock::new();
    C.get_or_init(|| Client::builder().build().unwrap())
}

fn url(path: &str) -> String {
    format!("{}{}", config().api_base_url, path)
}

pub async fn call<T, B>(method: Method, path: &str, body: Option<&B>)
    -> Result<T, HttpError>
where
    T: DeserializeOwned + Default,
    B: Serialize + ?Sized,
{
    let mut req: RequestBuilder = client()
        .request(method, url(path))
        .fetch_credentials_include();   // IMPORTANT: sends pds_session cookie

    if let Some(b) = body {
        req = req.header("content-type", "application/json").json(b);
    }

    let resp = req.send().await.map_err(HttpError::network)?;
    parse(resp).await
}

async fn parse<T: DeserializeOwned + Default>(resp: Response) -> Result<T, HttpError> {
    let status = resp.status();
    if status == StatusCode::NO_CONTENT {
        return Ok(T::default());                // () implements Default for unit responses
    }
    let text = resp.text().await.unwrap_or_default();
    if !status.is_success() {
        let err: ApiError = serde_json::from_str::<Envelope>(&text)
            .map(|e| e.error)
            .unwrap_or_else(|_| ApiError {
                code: "unknown".into(),
                message: if text.is_empty() { format!("HTTP {}", status) } else { text },
                details: None,
            });
        return Err(HttpError { status: status.as_u16(), error: err });
    }
    serde_json::from_str(&text).map_err(|e| HttpError::decode(status.as_u16(), e))
}
why fetch_credentials_include()
Browsers default to SameSite=Lax behaviour for fetch without that flag. In dev (Vite on :5173 talking to axum on :8080) that means the pds_session cookie is dropped and every authenticated call returns 401. Setting it once on the client builder isn't possible in the WASM backend — it's per-request — so we set it in every call.

Error type

// crates/web/src/api/error.rs — mirrors server's error envelope
#[derive(Deserialize, Clone, Debug)]
pub struct ApiError {
    pub code: String,
    pub message: String,
    #[serde(default)]
    pub details: Option<serde_json::Value>,
}

#[derive(Clone, Debug)]
pub struct HttpError {
    pub status: u16,
    pub error: ApiError,
}

impl HttpError {
    pub fn code(&self) -> &str { &self.error.code }
    pub fn message(&self) -> &str { &self.error.message }
    pub fn is_forbidden(&self) -> bool { self.status == 403 }
    pub fn network(e: reqwest::Error) -> Self { /* ... */ }
    pub fn decode(status: u16, _: serde_json::Error) -> Self { /* ... */ }
}

Endpoint helpers

One file per surface. Each function is a thin wrapper around call::<_, _>(Method, path, body). Total surface ≈ same as today's web/src/api/index.ts. 

// crates/web/src/api/auth.rs
pub async fn setup_status() -> Result<SetupStatus, HttpError> {
    call(Method::GET, "/api/v1/setup/status", None::<&()>).await
}
pub async fn create_first_admin(input: &CreateAdminInput) -> Result<UserResp, HttpError> {
    call(Method::POST, "/api/v1/setup/admin", Some(input)).await
}
pub async fn me()       -> Result<UserPublic, HttpError> {
    call(Method::GET,  "/api/v1/auth/me", None::<&()>).await
}
pub async fn login(input: &LoginInput) -> Result<UserPublic, HttpError> {
    call::<UserResp, _>(Method::POST, "/api/v1/auth/login", Some(input))
        .await.map(|r| r.user)
}
pub async fn register(input: &RegisterInput) -> Result<UserPublic, HttpError> { /* ... */ }
pub async fn logout()  -> Result<(), HttpError> {
    call(Method::POST, "/api/v1/auth/logout", None::<&()>).await
}

Using the client from a component

#[component]
fn Dashboard() -> Element {
    let filter = use_signal(|| FilterTab::All);
    let query  = use_signal::<String>(String::new);

    // use_resource is the SWR-style hook — re-fires when its dependencies change.
    let snippets = use_resource(move || async move {
        api::snippets::list(ListOpts { limit: Some(50), ..Default::default() }).await
    });

    let rendered = match &*snippets.read_unchecked() {
        Some(Ok(list))  => render_table(list, filter, query),
        Some(Err(e))    => rsx! { Error { message: e.message() } },
        None            => rsx! { "loading…" },
    };
    rsx! { Shell { {rendered} } }
}
why use_resource and not use_effect
use_resource is the idiomatic Dioxus 0.7 fetch primitive — it tracks the closure's reactive reads as dependencies, re-runs the future when they change, and gives you Suspense-friendly Option<Result<…>> state without manual loading flags. It's the right replacement for the imperative onMounted(load) patterns in the old views.

Request cancellation

The Vue code doesn't cancel in-flight requests; it just races and keeps the latest. We do the same in Dioxus — use_resource automatically discards the result of a previous future when its dependencies change, so we don't need explicit AbortController-style plumbing for the editor view either.

Smoke-test plan

Before any page work lands, we land the client + a single end-to-end smoke test that:

  1. boot() hits /api/v1/setup/status + /auth/me
  2. Registers a new user against a temp DB
  3. Logs in, lists snippets, creates a snippet, fetches it, deletes it
  4. Logs out and confirms the cookie is gone

Run with wasm-bindgen-test (headless Chrome via wasm-pack test) so it sits beside the rest of the workspace's cargo test --workspace.
