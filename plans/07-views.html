View pages

Three read-only pages for the slug routes: /c/:slug (code), /m/:slug (markdown), /h/:slug (sandboxed html). Same header layout, same action row, same modals.

Common loader

All three views start with the same fetch. A tiny shared use_snippet hook keeps the code DRY: 

fn use_snippet(slug: String) -> Resource<Result<Snippet, HttpError>> {
    let slug = slug.clone();
    use_resource(move || {
        let slug = slug.clone();
        async move { api::snippets::get(&slug).await }
    })
}

If the type returned by the server doesn't match the route, we redirect to the right prefix — same fallback as today: 

use_effect(move || {
    if let Some(Ok(s)) = snippet.read_unchecked().as_ref() {
        if s.r#type != SnippetType::Html {
            navigator().replace(Route::for_snippet(s));
        }
    }
});

ViewCode

Calls the same highlight worker as the editor (different code path: explicit-language-or-detect, no append cache because the body never changes after load). Renders a flex [gutter | pre]: 

#[component]
pub fn ViewCode(slug: String) -> Element {
    let snippet = use_snippet(slug);
    let mut hl = use_signal(HighlightCache::default);

    use_effect(move || {
        if let Some(Ok(s)) = snippet.read_unchecked().as_ref() {
            if s.r#type == SnippetType::Code {
                highlight::request(&s.body, SnippetType::Code, move |r| hl.write().on_worker_reply(r));
            }
        }
    });

    let render = match snippet.read_unchecked().as_ref() {
        Some(Ok(s)) if s.r#type == SnippetType::Code =>
            rsx! { CodeBody { snippet: s.clone(), html: hl.read().html.clone() } },
        Some(Err(e)) => rsx! { div { class: "text-danger", "{e.message()}" } },
        _ => rsx! {},
    };
    rsx! { Shell { {render} } }
}

The gutter is one div per logical newline, same as the Vue version. line_count is body.split('\n').count().max(1).

Pure synchronous render via pulldown-cmark; no worker. The article goes into a .md-preview container with the same CSS rules as today (see 10).

ViewHtml — sandboxed iframe + size reporter

This is the gnarly one because the iframe auto-sizes both dimensions based on postMessage events from the server-injected size reporter. Translating each piece:

concernsame as today?how it changes
load via src=/h/:slug/rawyesiframe attribute is identical, server still injects the reporter
sandbox attrsyessandbox="allow-scripts allow-popups" + referrerpolicy="no-referrer"
height & width stateyestwo Signal<u32>s with the same SIZE_SLACK / HYSTERESIS logic
window message listeneryeswasm-bindgen Closure + add_event_listener_with_callback
"only trust msgs from our iframe"yese.source() == iframe.content_window() comparison
horizontal scroll wrapperyesdiv.overflow-x-auto as today
 
#[component]
pub fn ViewHtml(slug: String) -> Element {
    let snippet = use_snippet(slug);
    let mut h = use_signal(|| 400u32);
    let mut w = use_signal::<Option<u32>>(|| None);
    let mut iframe_ref = use_signal::<Option<HtmlIFrameElement>>(|| None);

    use_effect(move || {
        let closure = Closure::<dyn FnMut(MessageEvent)>::new(move |e: MessageEvent| {
            let Some(ifr) = iframe_ref.read().as_ref() else { return; };
            if e.source() != ifr.content_window().map(|w| w.into()) { return; }
            let data: SizeMsg = serde_wasm_bindgen::from_value(e.data()).unwrap_or_default();
            if data.kind == "pastedev:size" {
                if let Some(hh) = data.height { commit_h(&mut h, hh); }
                if let Some(ww) = data.width  { commit_w(&mut w, ww); }
            }
        });
        window()
            .add_event_listener_with_callback("message", closure.as_ref().unchecked_ref())
            .ok();
        closure.forget();   // listener lives for the page lifetime
    });

    rsx! {
        Shell {
            // ... header + meta + actions row, same layout ...
            div { class: "overflow-x-auto",
                iframe {
                    onmounted: move |c| iframe_ref.set(Some(downcast(c))),
                    src: "{raw_url}",
                    sandbox: "allow-scripts allow-popups",
                    referrerpolicy: "no-referrer",
                    scrolling: "no",
                    title: "{title}",
                    class: "block bg-white border border-border rounded-sm",
                    style: format!(
                        "height:{}px;width:{};min-width:100%",
                        h(), w().map(|x| format!("{}px", x)).unwrap_or_else(|| "100%".into()),
                    ),
                }
            }
        }
    }
}
do not re-implement the size reporter
The server already injects the size-reporter JS into every /h/:slug/raw response (see crates/server/src/snippets/handlers.rs HTML_SIZE_REPORTER). Don't try to inject it from the client — the iframe is null-origin, the SPA can't reach into it.

Actions row — clipboard, edit, delete

Three small interactions, each mapped:

actionvuedioxus
copy link navigator.clipboard.writeText(snippet.url) web_sys::window().navigator().clipboard().write_text(&url) (Promise → spawn(async move { ... }.await))
copy raw same, with snippet.body same — gated behind a copied signal + a 1.5s timeout
edit RouterLink :to="`/?edit=${slug}`" Link { to: Route::Editor { edit: Some(slug.clone()) } }
delete Modal → api.deleteSnippet → router.replace('/dashboard') Modal component, api::snippets::delete, navigator().replace(Route::Dashboard{})

"Can I edit this?"

Same predicate: auth.user.username == snippet.owner.username. Wrapped in a tiny helper: 

fn can_edit(s: &Snippet, auth: &AuthState) -> bool {
    auth.user.read().as_ref().is_some_and(|u| u.username == s.owner.username)
}

Fallback / 404

When api::snippets::get returns 404 not_found we show the same minimal error line we do today (no redirect — the slug might come back if the user passed it around). The NotFound catch-all route handles entirely unknown URLs; it renders inside Shell with a single "this page doesn't exist" line, matching the spec.
