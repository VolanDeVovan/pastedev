Parity checklist

Every user-visible behaviour the Vue SPA does today. The cutover PR lands when every box is ticked. Each row links back to the chapter that owns it.

Legend

keep behaves identically — same UX, no visible change.   rewrite reimplemented in Rust; should behave the same but new code.   new didn't exist before this migration.   drop intentionally removed during migration.

Boot & routing → 04

  • ☐ SPA mounts at <div id="main"> after WASM hydration. rewrite
  • ☐ Reads <script id="pastedev-config"> on boot. keep
  • ☐ Calls /api/v1/setup/status + /auth/me in parallel before first navigation completes. keep
  • ☐ Setup gate: while needs_setup, all routes except /setup and /status redirect to /setup. keep
  • ☐ Signed-out users hitting protected routes redirect to /signin?next=…. keep
  • ☐ Signed-in users on /signin or /register redirect to /. keep
  • ☐ pending / rejected users get parked on the matching page until status changes. keep
  • ☐ Admins see /admin; non-admins get bounced from it to /. keep
  • ☐ Approved users only on /, /dashboard, /keys. keep
  • ☐ HTML5 history (no hash); browser back/forward works. keep

Shell + nav → 09

  • ☐ 52-pixel top bar with the pastedev wordmark linking to /. keep
  • ☐ Desktop nav shows new · my snippets · api keys · (admin) for approved/admin users only. keep
  • ☐ Active route gets an accent-color underline on its nav link. keep
  • ☐ Top-right shows username + sign-out, or sign-in/register CTA. keep
  • ☐ Mobile bottom tab bar appears for approved users only; 4 cols if admin, 3 otherwise. keep
  • ☐ Active tab gets a 24px accent stripe at the top of the cell. keep
  • ☐ Safe-area inset padding on mobile (env(safe-area-inset-bottom)). keep

Editor → 06

  • ☐ Three type tabs (code · md · html); locked when editing an existing snippet. keep
  • ☐ Filename input, decorative expires:never / visibility:public chips, byte counter, publish button. keep
  • ☐ Counter turns red above 1 MB; publish disabled. keep
  • ☐ Cmd/Ctrl+Enter publishes. keep
  • ☐ Inline highlight overlay for code & html; no white flash on keystroke. rewrite
  • ☐ Append / backspace / kind-flip / paste-replace fast paths preserved. rewrite
  • ☐ Line-number gutter on code & html; scroll-synced via translateY. keep
  • ☐ Markdown mode shows split-pane preview (desktop) / stacked (mobile), 120 ms debounce. keep
  • ☐ Detected language shows in statusline; hl off · large file banner appears above the worker skip threshold. keep
  • ☐ Edit mode prefills from ?edit=<slug>; PATCH on save. keep
  • ☐ 403 on publish refreshes auth state. keep

View pages → 07

  • ☐ /c/:slug renders with code body, gutter, hljs highlight. keep
  • ☐ /m/:slug renders the markdown article via .md-preview. keep
  • ☐ /h/:slug embeds via sandboxed iframe; height & width track the size-reporter messages. keep
  • ☐ Header actions: copy link / copy raw / open ↗ / edit / delete (where allowed). keep
  • ☐ Delete prompts via the danger modal. keep
  • ☐ Type-mismatched URL redirects to the right prefix. keep
  • ☐ Per-snippet OG/Twitter meta tags inherited from server-rendered shell. keep

Auth pages → 08

  • ☐ Sign-in form + ?next= redirect. keep
  • ☐ Register form with reason (≥ 10 chars); redirects to /pending. keep
  • ☐ Pending page with three pulsing dots; polls every 10 s; redirects when status changes. keep
  • ☐ Rejected page is terminal; only sign-out. keep
  • ☐ Sign-out clears the cookie and navigates to /signin. keep

Setup wizard → 08

  • ☐ Step 1 (env check) renders the checks list with status dots; polls every 2 s. keep
  • ☐ Continue is disabled until all checks are ok. keep
  • ☐ Step 2 (admin form): username + email + password + auto-approve callout. keep
  • ☐ Submit creates the first admin; redirects to /. keep
  • ☐ Mobile shows the step strip; desktop shows the sidebar. keep

Status page → 08

  • ☐ Renders the same checks API the server returns. keep
  • ☐ Adaptive polling: 5 s when all ok, 15 s when not. keep
  • ☐ "Refresh now" button forces an immediate poll. keep
  • ☐ Overall status (ok / warn / err) drives the eyebrow color. keep

  • ☐ Per-user card: avatar, username, email, request reason, requested-ago, IP, status & role chips. keep
  • ☐ Avatar hue derived from username (stable across refreshes). keep
