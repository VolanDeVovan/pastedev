Editor & highlight

The editor is 427 lines of Vue and most of the trickiness lives in the keep-colors-while-typing cache. The Dioxus port preserves the strategy line-for-line — only the language and host framework change.

Why the editor is special

The current implementation rejects every off-the-shelf editor library (Monaco, CodeMirror 6) because none of them auto-detect across hljs's ~200 languages. So the editor is a plain <textarea> with an absolutely-positioned <pre> sitting underneath, both sharing the same padding/box. The <pre> shows hljs-highlighted html via v-html; the textarea has color: transparent and forwards the caret. Scroll position is synced manually.

┌─────────────────────────── editor body ────────────────────────────┐ │ │ │ ┌─── line gutter ───┐ ┌──────────── stacked layers ───────────┐ │ │ │ 1 │ │ <pre> (overlay) color spans │ │ ← v-html │ │ 2 │ │ │ │ │ (hljs output) │ │ … │ │ <textarea> transparent text │ │ ← caret + input │ └───────────────────┘ └─────────────────────────────────────-─┘ │ │ │ └────────────────────────────────────────────────────────────────────┘ │ │ debounced postMessage (150 ms) ▼ ┌─ highlight.worker.js ──────────┐ │ hljs.highlightAuto / highlight │ └────────────────────────────────┘ ▲ │ Reply { id, html, language, truncated } ▼ main thread caches: cachedBody, cachedHtml fast-path on next keystroke (append / backspace / other)

The cache and fast paths — kept as-is

The Vue version has three fast paths so the colored html doesn't flash to plain text on every keystroke while the worker reply is in flight. Lifting them verbatim into Rust:

casewhat we paint immediately
cold start (first render, edit-mode load)escape_html(body) — plain but visible, never blank
append (most keystrokes)cached html + escaped tail
backspacetruncate cached html at new char count, re-close any open spans
middle-edit / paste-replaceescape the whole body for one frame; worker repaints
kind flipinvalidate cache (same body would tokenize differently as code vs html)
cold-start prepaint matters
Because the textarea uses color: transparent, the overlay is what the user actually sees. Without the cold-start escape_html step, opening /?edit=<slug> shows a blank canvas until the worker's first reply lands (50–250 ms depending on body size). Always seed HighlightCache.html with the escaped body on the first paint; the worker reply replaces it whenever it arrives.

The EditorPage component

// crates/web/src/editor/mod.rs
#[component]
pub fn EditorPage(edit: Option<String>) -> Element {
    let auth   = use_auth();
    let toast  = use_toast();
    let nav    = navigator();

    let mut kind       = use_signal(|| SnippetType::Code);
    let mut body       = use_signal(String::new);
    let mut name       = use_signal(String::new);
    let mut editing    = use_signal::<Option<String>>(|| None);
    let mut error      = use_signal::<Option<String>>(|| None);
    let mut submitting = use_signal(|| false);

    // Stateful highlighter: keeps cachedBody/cachedHtml across renders.
    let mut hl = use_signal(HighlightCache::default);

    // If ?edit=<slug> changes (back/forward nav, manual URL edit), refetch.
    // use_resource — not use_effect+spawn — gives free cancellation: when
    // the dependency (edit) changes, the previous future's result is
    // discarded rather than racing the new one. The same pattern as the
    // view pages in chapter 07.
    let loaded = use_resource(move || {
        let slug = edit.clone();
        async move {
            match slug {
                Some(s) => api::snippets::get(&s).await.map(Some),
                None    => Ok(None),
            }
        }
    });

    // When a load completes, push it into the form signals exactly once.
    // Using use_effect on a memo of the Resource's value keeps signal
    // updates out of the render pass — Dioxus is strict about that.
    let loaded_snapshot = use_memo(move || loaded.read_unchecked().clone());
    use_effect(move || {
        match loaded_snapshot() {
            Some(Ok(Some(s))) => {
                editing.set(Some(s.slug.clone()));
                kind.set(s.kind);
                body.set(s.body.clone());
                name.set(s.name.clone().unwrap_or_default());
                // Cold-start prepaint: seed the overlay with escaped body so
                // there's no blank frame between mount and the first worker reply.
                hl.write().prepaint_escaped(&s.body);
            }
            Some(Err(e)) => error.set(Some(e.message().into())),
            _ => {}
        }
    });

    // Reactively recompute overlay when body/kind change.
    let overlay_html = use_memo(move || {
        hl.write().recompute(&body.read(), kind())
    });

    // Fire-and-forget worker call on every body/kind change (debounced inside).
    use_effect(move || {
        highlight::request(&body.read(), kind(), move |reply| {
            hl.write().on_worker_reply(reply);
        });
    });

    let submit = move |_| spawn(async move {
        if body.read().is_empty() { error.set(Some("body is empty".into())); return; }
        submitting.set(true);
        let result = match editing() {
            Some(slug) => api::snippets::patch(&slug, &PatchInput {
                body: Some(body()), name: name_or_null(name()),
            }).await.map(|s| s.url),
            None => api::snippets::create(&CreateSnippetInput {
                r#type: kind(), name: trim_or_none(name()), body: body(),
            }).await.map(|s| s.url),
        };
        submitting.set(false);
        match result {
            Ok(u)  => nav.replace(path_of(&u)),
            Err(e) => {
                if e.is_forbidden() { auth.refresh_me().await; }
                error.set(Some(e.message().into()));
                toast.error(e.message());
            }
        }
    });

    rsx! {
        Shell {
            EditorToolbar { kind, name, editing, submitting, on_submit: submit }
            EditorBody {
                kind, body, overlay_html,
                inline_highlight: matches!(kind(), SnippetType::Code | SnippetType::Html),
            }
            EditorStatusline {
                size_bytes: body.read().len(),
                detected_lang: hl.read().language.clone(),
                truncated: hl.read().truncated,
                error: error.read().clone(),
            }
        }
    }
}

Cmd/Ctrl + Enter

Same shortcut, but Dioxus's onkeydown takes a typed KeyboardEvent instead of a raw DOM event. Cleaner — no .preventDefault() threading through string keys. 

onkeydown: move |e: KeyboardEvent| {
    if (e.modifiers().meta() || e.modifiers().ctrl())
        && e.key() == Key::Enter
    {
        e.prevent_default();
        submit(());
    }
},

Scroll sync: textarea ↔ overlay ↔ gutter

We need raw DOM access to read scrollTop/scrollLeft and mutate style.transform on the gutter. Dioxus gives us element references via onmounted + a stored web_sys::HtmlElement: 

let mut ta_el      = use_signal::<Option<HtmlTextAreaElement>>(|| None);
let mut overlay_el = use_signal::<Option<HtmlElement>>(|| None);
let mut gutter_el  = use_signal::<Option<HtmlElement>>(|| None);

let sync_scroll = move |_| {
    if let (Some(ta), Some(ov), Some(g)) =
        (ta_el().as_ref(), overlay_el().as_ref(), gutter_el().as_ref())
    {
        ov.set_scroll_top(ta.scroll_top());
        ov.set_scroll_left(ta.scroll_left());
        let _ = g.style().set_property(
            "transform",
            &format!("translateY({}px)", -ta.scroll_top()),
        );
    }
};

rsx! {
    textarea {
        onmounted: move |c| ta_el.set(Some(downcast::<HtmlTextAreaElement>(c))),
        onscroll: sync_scroll,
        oninput:  move |e| { body.set(e.value()); sync_scroll(()); },
        // ... value, classes, placeholder
    }
}

Markdown preview

Same 120 ms debounce. pulldown-cmark renders to a String of html on the main thread (it's fast enough for bodies under a megabyte). We mount it with dangerous_inner_html: 

use pulldown_cmark::{Parser, Options, html};

pub fn render_markdown(src: &str) -> String {
    let mut opts = Options::empty();
    opts.insert(Options::ENABLE_TABLES);
    opts.insert(Options::ENABLE_FOOTNOTES);
    opts.insert(Options::ENABLE_STRIKETHROUGH);
    opts.insert(Options::ENABLE_TASKLISTS);
    opts.insert(Options::ENABLE_SMART_PUNCTUATION);
    let parser = Parser::new_ext(src, opts);
    let mut out = String::with_capacity(src.len() * 2);
    html::push_html(&mut out, parser);
    out
}

Fenced code blocks get hljs treatment by post-processing the rendered html on a second pass — same approach as the server's pulldown-cmark integration: emit <pre><code class="language-…">…</code></pre> and let the css rule decorate them. We do not enable Options::ENABLE_HTML — that's the XSS vector.

The worker — stays JS

No port to Rust. We keep web/src/workers/highlight.worker.ts verbatim (rename to .js after compiling, or keep TS and compile via esbuild), put it under crates/web/assets/highlight.worker.js, and let dx copy it into the dist via the asset!() macro:

we considered porting to Rust + did the math
  • syntect — TextMate-grammar based; requires explicit language. We'd lose auto-detect.
  • tree-sitter-highlight — needs one wasm grammar per language. ~200 of those is 30+ MB.
  • ts_rs / hljs port — no canonical maintained port; the heuristic relies on JS regex behaviour.
Keeping hljs in JS is ~120 kB gz. Replacing it would either lose capability or balloon the bundle. The interop boundary is one postMessage in each direction; not worth eliminating.

Above the snippet limit

The current code has two ceilings: the worker skips at > 4 MB (returns plain escaped html with truncated=true), and the composable skips before round-tripping at > ~1.1 MB. We keep both thresholds; the statusline shows hl off · large file.

Mobile considerations

The toolbar wraps on narrow viewports; the type-tabs row sits above the filename row when both don't fit. Dioxus has no flex-wrap primitive — it's all CSS classes — so the mobile/desktop split is the same Tailwind classes the Vue version uses (md:flex-nowrap, etc.). See 10.
